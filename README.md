# Передбачення 

Статичний вебпроєкт українською мовою для випадкових передбачень у стилі mystic minimal.  
Працює без бекенду і без фреймворків: тільки `HTML`, `CSS`, `JavaScript` і `JSON`.

## Огляд

Проєкт показує карту-передбачення, дозволяє:
- витягувати нові передбачення
- зберігати їх локально
- копіювати текст
- ділитися посиланням або PNG-карткою
- відкривати тематичні колоди
- автоматично підміняти дефолтну колоду за календарною датою

## Можливості

- Одна головна сторінка з картою-передбаченням.
- Окрема сторінка тематичних колод.
- Окрема сторінка збережених передбачень.
- Темна і світла тема.
- Локальне збереження через `localStorage`.
- PNG-експорт карти.
- Автоматичні сезонні або святкові колоди.
- Кастомний фон рубашки для кожної колоди.
- Приховані колоди, які працюють лише по даті, без показу в галереї.

## Архітектура

- `script.js` лише піднімає застосунок.
- `js/app.js` координує UI, стани, переходи між view і вибір поточної колоди.
- `js/config/deck-catalog.js` є джерелом правди для всіх колод.
- `js/config/decks.js` обирає активну колоду на основі каталогу і дат.
- `js/services/fortune-store.js` завантажує і кешує тексти колод.
- `js/services/saved-fortunes.js` ізолює роботу з `localStorage`.
- `js/services/share-image.js` генерує PNG-картку для поширення.
- `js/ui/back-design.js` рендерить іконки на рубашці карти.
- `js/ui/deck-appearance.js` підставляє кастомний фон рубашки.
- `js/ui/deck-gallery.js` будує сторінку тематичних колод.

## Запуск локально

Потрібен простий HTTP-сервер, бо `fetch` не працює через `file://`.


```text
http://localhost:8080
```

## Як працюють колоди

Кожна колода описується в `js/config/deck-catalog.js`.

Колода може містити:
- JSON з передбаченнями
- бейдж
- назву в галереї
- назву на рубашці
- короткий опис
- `priority` для конфлікту кількох календарних колод
- `galleryOrder` для порядку в галереї
- прапорець показу в галереї
- діапазон дат автопідключення
- дизайн рубашки
- кастомний фон рубашки

## Як додати нову колоду

### 1. Створи JSON з текстами

Приклад:

```text
data/fortunes_halloween.json
```

Формат:

```json
[
  { "text": "У тиші вечора знайдеться щось кумедно підозріле." },
  { "text": "Сьогодні дивна дрібниця захоче привернути твою увагу." }
]
```

### 2. Додай запис у каталог

Файл: `js/config/deck-catalog.js`

```js
halloween: buildDeck({
  id: "halloween",
  file: "fortunes_halloween.json",
  badge: "🎃",
  name: "Halloween",
  title: "Haunt",
  description: "Трохи темна, трохи іронічна колода для кінця жовтня.",
  priority: 80,
  galleryOrder: 40,
  showInGallery: true,
  activeFrom: { month: 9, day: 25 },
  activeTo: { month: 9, day: 31 },
  backDesign: {
    layout: "diagonal",
    icons: ["✦", "☾", "🕯", "✶"],
    background: {
      dark: "linear-gradient(180deg, rgba(33, 18, 12, 0.98), rgba(18, 10, 7, 0.98))"
    }
  }
})
```

## Оформлення рубашки

Поле `backDesign` підтримує:

```js
backDesign: {
  layout: "center",
  icons: ["✦"],
  background: {}
}
```

або

```js
backDesign: {
  layout: "diagonal",
  icons: ["☾", "✦", "☽", "✧"],
  background: {
    dark: "linear-gradient(...)",
    light: "linear-gradient(...)"
  }
}
```

Пояснення:
- `layout: "center"`: одна велика іконка по центру
- `layout: "diagonal"`: повторюваний декоративний патерн
- `icons`: набір символів для рубашки
- `background.dark`: фон рубашки в темній темі
- `background.light`: фон рубашки в світлій темі

Якщо `background` порожній, використовується стандартний фон сайту.

### Unicode-символи для монохромних рубашок

## Автоматичне підключення за датою

Якщо колода має автоматично ставати активною замість дефолтної, достатньо задати:

```js
activeFrom: { month: 9, day: 25 },
activeTo: { month: 9, day: 31 }
```

Якщо на одну дату підходить кілька колод, перемагає колода з більшим `priority`:

```js
priority: 100
```

Важливо:
- місяці в JavaScript нумеруються з нуля
- `0` — січень
- `9` — жовтень
- `11` — грудень
- якщо `activeFrom` і `activeTo` порожні, колода не активується автоматично
- якщо `priority` не заданий, використовується `0`

## Галерея тематичних колод

Сторінка `Колоди` будується автоматично з `deck-catalog.js`.

Щоб колода показувалась у галереї:

```js
showInGallery: true
```

Щоб колода працювала лише по календарю і не показувалась вручну:

```js
showInGallery: false
```

Це корисно для:
- одноденних святкових колод
- специфічних колод, які не треба відкривати вручну

Порядок карток у галереї керується окремо:

```js
galleryOrder: 40
```

Важливо:
- менше число `galleryOrder` = вище в списку
- якщо `galleryOrder` не заданий, колода піде після впорядкованих вручну
- `priority` не впливає на порядок у галереї

Фактичне правило сортування:
- спочатку всі колоди з меншим `galleryOrder`
- якщо `galleryOrder` однаковий, далі сортування за `name`
- якщо `galleryOrder` не заданий, використовується кінець списку

Приклад:

```js
default: buildDeck({
  galleryOrder: 0
}),
winter: buildDeck({
  galleryOrder: 20
}),
xmas: buildDeck({
  galleryOrder: 30
}),
hiddenHoliday: buildDeck({
  showInGallery: false
})
```

Результат у галереї:
- `default`
- `winter`
- `xmas`
- `hiddenHoliday` не показується взагалі

## Промти для генерації текстів

### Нова колода

```text
Згенеруй тематичну колоду передбачень для сайту.

ВВІД:
Тема: [ВСТАВИТИ ТЕМУ, наприклад: Хелловін / Різдво / Любов / Літо]
Очікування: [ВСТАВИТИ 1-2 РЕЧЕННЯ — яка атмосфера, який характер колоди]
Баланс: [ВСТАВИТИ, наприклад: 20% гумор / 80% атмосфера]
Кількість передбачень: [ВСТАВИТИ ЧИСЛО, наприклад: 60 / 100 / 120]

---

КРОК 1 — Визнач параметри колоди самостійно.

На основі теми та очікувань придумай і виведи:
- id: коротке латинське слово (наприклад: easter, christmas, love)
- file: fortunes_[id].json
- badge: один емодзі що передає тему
- name: назва колоди українською (1-2 слова)
- title: одне коротке слово для рубашки карти (укр або англ)
- description: 1 речення — атмосферний короткий опис колоди
- priority: число для пріоритету при перетині кількох календарних колод
- galleryOrder: число для позиції в галереї
- showInGallery: true
- activeFrom / activeTo: логічний діапазон дат для теми
  (місяці в JS з нуля: 0=січень, 11=грудень)
- backDesign:
  - layout: "center" або "diagonal"
  - icons: 2–4 монохромних символи з набору:
    ✦ ✧ ✶ ✷ ✺ ✹ ☾ ☽ ☼ ☻ ❤ ♡ ❥ ❄ ✵ ✴ ◈ ◇ ◆
  - background.dark і background.light — градієнти під тему

Виведи параметри у форматі:
id: ...
file: ...
badge: ...
name: ...
title: ...
description: ...
priority: N
galleryOrder: N
showInGallery: ...
activeFrom: { month: N, day: N }
activeTo: { month: N, day: N }
backDesign: { layout: "...", icons: [...], background: { dark: "...", light: "..." } }


---

КРОК 2 — Згенеруй рівно [КІЛЬКІСТЬ] передбачень українською.

Стиль:
- Живі, несподівані, з характером — не "мудрі цитати"
- Різноманітний ритм: короткі рубані, довші з поворотом
- Завжди від другої особи — "ти", "тебе", "тобі"
- Тема через деталі й ситуації, а не пряму назву
- Баланс гумору згідно з вводом

Заборонено:
- Повчальний тон, філософські узагальнення
- Образи: шлях, доля, Всесвіт, паросток, коло, призначення
- Однаковий ритм у всіх реченнях
- Листівка, тост, мотиваційний постер
- Повтори образів або конструкцій

Формат — тільки JSON, рівно [КІЛЬКІСТЬ] об'єктів:
[
  { "text": "..." },
  { "text": "..." }
]

---

КРОК 3 — Виведи блок коду для deck-catalog.js.

[id]: buildDeck({
  id: "...",
  file: "...",
  badge: "...",
  name: "...",
  title: "...",
  description: "...",
  priority: N,
  galleryOrder: N,
  showInGallery: true/false,
  activeFrom: { month: N, day: N },
  activeTo: { month: N, day: N },
  backDesign: {
    layout: "...",
    icons: ["..."],
    background: {
      dark: "linear-gradient(...)",
      light: "linear-gradient(...)"
    }
  }
})

---

Виведи всі три кроки по черзі. Нічого зайвого між ними.

```

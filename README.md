# Передбачення 

Статичний вебпроєкт українською мовою для випадкових передбачень у стилі mystic minimal.  
Працює без бекенду і без фреймворків: тільки `HTML`, `CSS`, `JavaScript` і `JSON`.

## Огляд

Проєкт показує карту-передбачення, дозволяє:
- витягувати нові передбачення
- зберігати їх локально
- копіювати текст
- ділитися посиланням або PNG-карткою
- відкривати тематичні колоди
- автоматично підміняти дефолтну колоду за календарною датою

## Можливості

- Одна головна сторінка з картою-передбаченням.
- Окрема сторінка тематичних колод.
- Окрема сторінка збережених передбачень.
- Темна і світла тема.
- Локальне збереження через `localStorage`.
- PNG-експорт карти.
- Автоматичні сезонні або святкові колоди.
- Кастомний фон рубашки для кожної колоди.
- Приховані колоди, які працюють лише по даті, без показу в галереї.

## Архітектура

- `script.js` лише піднімає застосунок.
- `js/app.js` координує UI, стани, переходи між view і вибір поточної колоди.
- `js/config/deck-catalog.js` є джерелом правди для всіх колод.
- `js/config/decks.js` обирає активну колоду на основі каталогу і дат.
- `js/services/fortune-store.js` завантажує і кешує тексти колод.
- `js/services/saved-fortunes.js` ізолює роботу з `localStorage`.
- `js/services/share-image.js` генерує PNG-картку для поширення.
- `js/ui/back-design.js` рендерить іконки на рубашці карти.
- `js/ui/deck-appearance.js` підставляє кастомний фон рубашки.
- `js/ui/deck-gallery.js` будує сторінку тематичних колод.

## Запуск локально

Потрібен простий HTTP-сервер, бо `fetch` не працює через `file://`.

```bash
python -m http.server 8080
```

Після цього відкрий:

```text
http://localhost:8080
```

## Як працюють колоди

Кожна колода описується в `js/config/deck-catalog.js`.

Колода може містити:
- JSON з передбаченнями
- бейдж
- назву в галереї
- назву на рубашці
- короткий опис
- прапорець показу в галереї
- діапазон дат автопідключення
- дизайн рубашки
- кастомний фон рубашки

## Як додати нову колоду

### 1. Створи JSON з текстами

Приклад:

```text
data/fortunes_halloween.json
```

Формат:

```json
[
  { "text": "У тиші вечора знайдеться щось кумедно підозріле." },
  { "text": "Сьогодні дивна дрібниця захоче привернути твою увагу." }
]
```

### 2. Додай запис у каталог

Файл: `js/config/deck-catalog.js`

```js
halloween: buildDeck({
  id: "halloween",
  file: "fortunes_halloween.json",
  badge: "🎃",
  name: "Halloween",
  title: "Haunt",
  description: "Трохи темна, трохи іронічна колода для кінця жовтня.",
  showInGallery: true,
  activeFrom: { month: 9, day: 25 },
  activeTo: { month: 9, day: 31 },
  backDesign: {
    layout: "diagonal",
    icons: ["✦", "☾", "🕯", "✶"],
    background: {
      dark: "linear-gradient(180deg, rgba(33, 18, 12, 0.98), rgba(18, 10, 7, 0.98))"
    }
  }
})
```

## Оформлення рубашки

Поле `backDesign` підтримує:

```js
backDesign: {
  layout: "center",
  icons: ["✦"],
  background: {}
}
```

або

```js
backDesign: {
  layout: "diagonal",
  icons: ["☾", "✦", "☽", "✧"],
  background: {
    dark: "linear-gradient(...)",
    light: "linear-gradient(...)"
  }
}
```

Пояснення:
- `layout: "center"`: одна велика іконка по центру
- `layout: "diagonal"`: повторюваний декоративний патерн
- `icons`: набір символів для рубашки
- `background.dark`: фон рубашки в темній темі
- `background.light`: фон рубашки в світлій темі

Якщо `background` порожній, використовується стандартний фон сайту.

### Unicode-символи для монохромних рубашок

```text
✦ ✧ ✶ ✷ ✺ ✹
☾ ☽ ☼ ☻
❤ ♡ ❥
❄ ✵ ✴
◈ ◇ ◆
```

## Автоматичне підключення за датою

Якщо колода має автоматично ставати активною замість дефолтної, достатньо задати:

```js
activeFrom: { month: 9, day: 25 },
activeTo: { month: 9, day: 31 }
```

Важливо:
- місяці в JavaScript нумеруються з нуля
- `0` — січень
- `9` — жовтень
- `11` — грудень
- якщо `activeFrom` і `activeTo` порожні, колода не активується автоматично

## Галерея тематичних колод

Сторінка `Колоди` будується автоматично з `deck-catalog.js`.

Щоб колода показувалась у галереї:

```js
showInGallery: true
```

Щоб колода працювала лише по календарю і не показувалась вручну:

```js
showInGallery: false
```

Це корисно для:
- одноденних святкових колод
- специфічних колод, які не треба відкривати вручну

## Шерінг і превʼю

Для соцмереж сайт використовує:
- `og:title`
- `og:description`
- `og:image`
- `twitter:card`

Статичне превʼю лежить у `og-preview.svg`.

Якщо сайт буде на GitHub Pages, краще використовувати абсолютний URL до превʼю-зображення.

## Промти для генерації текстів

### Базові передбачення

```text
Згенеруй список коротких українських передбачень для атмосферного сайту з картами-передбаченнями.

Вимоги до стилю:
- Тон легкий, м’який, трохи загадковий, іноді грайливий.
- Не пиши надто серйозно, пафосно, повчально або офіційно.
- Не давай прямих порад у стилі “тобі треба”, “зроби”, “необхідно”.
- Не пиши як гороскоп із журналу, психологічна стаття або мотиваційний тренінг.
- Кожне передбачення має бути натяком, образом або делікатною підказкою, а не прямим твердженням.
- Формулювання мають залишати простір для уяви й особистої інтерпретації.
- Добре, якщо є легке відчуття магії, випадковості, знаків, збігів, настрою.
- Можна додавати побутові, теплі, трохи поетичні образи.
- Уникати банальностей типу “усе буде добре”, “успіх уже близько”, “доля подарує шанс”.
- Уникати надто темних, тривожних, агресивних або токсичних формулювань.
- Без езотеричного перевантаження і штампів.
- Кожен текст має звучати природно українською.

Формат:
- Згенеруй 100 варіантів.
- Кожен варіант: одне речення.
- Довжина: 8–18 слів.
- Без нумерації.
- Без повторів і дуже схожих конструкцій.
- Поверни результат у форматі JSON-масиву об'єктів:
[
  { "text": "..." },
  { "text": "..." }
]

Орієнтир по настрою:
- “Сьогодні дрібниця може виявитися знаком”
- “Випадкова розмова раптом набуде дивної ваги”
- “Щось звичне цього разу відкриється з іншого боку”

Але не копіюй ці фрази дослівно, а створи нові, живі й різноманітні варіанти.

Перед тим як віддати результат, перевір кожне передбачення і видали все, що:
- звучить занадто прямолінійно
- звучить як порада
- звучить як банальний мотиваційний афоризм
- повторює однаковий ритм або однакові образи
- містить штампи про долю, успіх, Всесвіт, призначення
```

### Іронічні передбачення

```text
Згенеруй список коротких українських іронічних передбачень для сайту з картами-передбаченнями.

Вимоги до стилю:
- Тон легкий, дотепний, трохи абсурдний, але не злий.
- Це має бути м’яка самоіронія, а не сарказм або образи.
- Передбачення мають звучати як кумедні натяки, а не як жарти “в лоб”.
- Уникай грубості, токсичності, принижень, політики, чорного гумору і крінжових мемних формулювань.
- Можна використовувати побутові ситуації, дивні збіги, легкий хаос дня, смішні деталі реального життя.
- Кожен текст має бути коротким, живим і таким, що його хочеться перечитати.
- Не використовуй однакову конструкцію у всіх реченнях.

Формат:
- Згенеруй 80 варіантів.
- Кожен варіант: одне речення.
- Довжина: 8–18 слів.
- Без нумерації.
- Поверни результат у форматі JSON-масиву:
[
  { "text": "..." }
]

Орієнтир:
- смішно, але делікатно
- дивно, але впізнавано
- іронічно, але тепло

Перед відправкою прибери все, що:
- звучить занадто грубо
- занадто схоже на стендап-жарт
- повторює одні й ті самі побутові теми
- не залишає місця для легкої фантазії
```

### Сезонні або святкові передбачення

```text
Згенеруй список коротких українських тематичних передбачень для спеціальної сезонної колоди сайту.

Тема колоди: [ВСТАВ ТУТ ТЕМУ, наприклад Halloween / Різдво / Літо / День закоханих]

Вимоги до стилю:
- Передбачення мають передавати атмосферу теми, але не бути надто буквальними.
- Тон м’який, образний, трохи магічний, іноді грайливий.
- Не пиши надто пафосно або як шаблонне святкове привітання.
- Не роби всі тексти однаково “листівковими”.
- Можна використовувати сезонні деталі: світло, запахи, погоду, вечір, речі, звуки, святкові асоціації.
- Кожне передбачення має залишати простір для інтерпретації, а не просто називати тему напряму.
- Тексти мають бути різноманітні за ритмом і образами.

Формат:
- Згенеруй 60 варіантів.
- Кожен варіант: одне речення.
- Довжина: 8–18 слів.
- Без нумерації.
- Поверни результат у форматі JSON-масиву:
[
  { "text": "..." }
]

Додаткова умова:
- приблизно 70% текстів мають бути атмосферними й красивими
- приблизно 30% можуть бути легшими, теплішими або трохи жартівливими

Перед відправкою видали все, що:
- занадто прямо називає свято в кожному рядку
- звучить як реклама, привітання або тост
- повторює однакові образи
- занадто серйозне або занадто банальне
```

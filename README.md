# Prediction

# Передбачення ✨ (UA)

Мінімальний статичний сайт українською мовою для випадкових передбачень у стилі **Mystic Minimal**.  
Без серверів, без фреймворків — лише чисті **HTML/CSS/JS** та JSON з даними.

## Функціонал
- Одна сторінка `index.html`.
- Кнопка **«Дай передбачення»** відкриває карту та показує випадковий текст.
- Повторні натискання прокручують карту через рубашку і показують нове передбачення.
- Компактний ряд дій дозволяє **зберегти**, **скопіювати**, **поділитися**, а також відкрити поширення в **Instagram**, **Facebook**, **X**, **Threads**.
- Збережені передбачення тримаються локально в `localStorage` і не губляться після перезавантаження сторінки.
- Для **Instagram** сайт копіює готовий текст для ручної вставки, бо прямого веб-шерингу туди немає.
- Є перемикач **світлої/темної теми**, за замовчуванням увімкнено темну.
- Дані зберігаються в `data/fortunes.json` як масив об’єктів `{ text }`.
- Усе працює клієнтськи через `fetch`.
- Підтримуються сезонні набори передбачень через окремі JSON-файли.
- Для красивого шерингу додано `Open Graph` / `Twitter Card` метатеги та статичне preview-зображення `og-preview.svg`.

## Красивий шерінг

Для прев’ю в соцмережах сайт використовує:
- `og:title`
- `og:description`
- `og:image`
- `twitter:card`

Статичне зображення для прев’ю лежить у файлі `og-preview.svg`.

Важливо:
- якщо сайт уже опублікований на GitHub Pages, для максимального результату в метатегах краще вказати абсолютний URL до preview-зображення
- текст конкретного передбачення не підставляється в `og:image`, бо сайт повністю статичний і без бекенду
- натомість користувач отримує красиве брендоване прев’ю сторінки плюс окремо може поділитися самим текстом через кнопки шерингу

## Запуск локально
Потребується будь-який простий HTTP-сервер (через `file://` `fetch` не працюватиме).

```bash
# Python 3
python -m http.server 8080
# Відкрийте у браузері:
# http://localhost:8080
```

## Як додати тематичний гороскоп на період

Якщо потрібно показувати окремий набір передбачень у конкретні дати, схема така:

1. Створіть новий JSON у папці `data/`, наприклад `data/fortunes_halloween.json`.
2. Формат файлу має лишатися таким самим: масив об'єктів виду `{ "text": "..." }`.
3. У [script.js](./script.js) в функції `getSeasonInfo()` додайте нову умову з потрібним діапазоном дат.
4. У цій умові поверніть:
   - `key` з унікальною назвою сезону
   - `url` до нового JSON
   - `badge` з короткою міткою, яка з'явиться на карті

Приклад для періоду з 25 по 31 жовтня:

```js
if (m === 9 && d >= 25 && d <= 31) {
  return {
    key: "halloween",
    url: new URL("./data/fortunes_halloween.json", window.location.href).toString(),
    badge: "🎃 Halloween"
  };
}
```

Пояснення:
- `m === 9` означає жовтень, бо місяці в JavaScript рахуються з нуля.
- `d >= 25 && d <= 31` задає потрібний період.
- якщо дата не підпадає під жодну спеціальну умову, сайт автоматично використовує `data/fortunes.json`.

Рекомендація:
- нову умову краще ставити вище за `default`, у тому ж стилі, що вже існуючі сезонні варіанти
- назву файла і шлях треба вказувати точно, інакше на GitHub Pages буде помилка завантаження

## Промти для генерації передбачень

Нижче готові промти, які можна використовувати для генерації нових наборів текстів. Вони спеціально налаштовані так, щоб передбачення не були занадто серйозними, прямолінійними або офіційними.

### 1. Базові передбачення

```text
Згенеруй список коротких українських передбачень для атмосферного сайту з картами-передбаченнями.

Вимоги до стилю:
- Тон легкий, м’який, трохи загадковий, іноді грайливий.
- Не пиши надто серйозно, пафосно, повчально або офіційно.
- Не давай прямих порад у стилі “тобі треба”, “зроби”, “необхідно”.
- Не пиши як гороскоп із журналу, психологічна стаття або мотиваційний тренінг.
- Кожне передбачення має бути натяком, образом або делікатною підказкою, а не прямим твердженням.
- Формулювання мають залишати простір для уяви й особистої інтерпретації.
- Добре, якщо є легке відчуття магії, випадковості, знаків, збігів, настрою.
- Можна додавати побутові, теплі, трохи поетичні образи.
- Уникати банальностей типу “усе буде добре”, “успіх уже близько”, “доля подарує шанс”.
- Уникати надто темних, тривожних, агресивних або токсичних формулювань.
- Без езотеричного перевантаження і штампів.
- Кожен текст має звучати природно українською.

Формат:
- Згенеруй 100 варіантів.
- Кожен варіант: одне речення.
- Довжина: 8–18 слів.
- Без нумерації.
- Без повторів і дуже схожих конструкцій.
- Поверни результат у форматі JSON-масиву об'єктів:
[
  { "text": "..." },
  { "text": "..." }
]

Орієнтир по настрою:
- “Сьогодні дрібниця може виявитися знаком”
- “Випадкова розмова раптом набуде дивної ваги”
- “Щось звичне цього разу відкриється з іншого боку”

Але не копіюй ці фрази дослівно, а створи нові, живі й різноманітні варіанти.

Перед тим як віддати результат, перевір кожне передбачення і видали все, що:
- звучить занадто прямолінійно
- звучить як порада
- звучить як банальний мотиваційний афоризм
- повторює однаковий ритм або однакові образи
- містить штампи про долю, успіх, Всесвіт, призначення
```

### 2. Іронічні передбачення

```text
Згенеруй список коротких українських іронічних передбачень для сайту з картами-передбаченнями.

Вимоги до стилю:
- Тон легкий, дотепний, трохи абсурдний, але не злий.
- Це має бути м’яка самоіронія, а не сарказм або образи.
- Передбачення мають звучати як кумедні натяки, а не як жарти “в лоб”.
- Уникай грубості, токсичності, принижень, політики, чорного гумору і крінжових мемних формулювань.
- Можна використовувати побутові ситуації, дивні збіги, легкий хаос дня, смішні деталі реального життя.
- Кожен текст має бути коротким, живим і таким, що його хочеться перечитати.
- Не використовуй однакову конструкцію у всіх реченнях.

Формат:
- Згенеруй 80 варіантів.
- Кожен варіант: одне речення.
- Довжина: 8–18 слів.
- Без нумерації.
- Поверни результат у форматі JSON-масиву:
[
  { "text": "..." }
]

Орієнтир:
- смішно, але делікатно
- дивно, але впізнавано
- іронічно, але тепло

Перед відправкою прибери все, що:
- звучить занадто грубо
- занадто схоже на стендап-жарт
- повторює одні й ті самі побутові теми
- не залишає місця для легкої фантазії
```

### 3. Сезонні або святкові передбачення

```text
Згенеруй список коротких українських тематичних передбачень для спеціальної сезонної колоди сайту.

Тема колоди: [ВСТАВ ТУТ ТЕМУ, наприклад Halloween / Різдво / Літо / День закоханих]

Вимоги до стилю:
- Передбачення мають передавати атмосферу теми, але не бути надто буквальними.
- Тон м’який, образний, трохи магічний, іноді грайливий.
- Не пиши надто пафосно або як шаблонне святкове привітання.
- Не роби всі тексти однаково “листівковими”.
- Можна використовувати сезонні деталі: світло, запахи, погоду, вечір, речі, звуки, святкові асоціації.
- Кожне передбачення має залишати простір для інтерпретації, а не просто називати тему напряму.
- Тексти мають бути різноманітні за ритмом і образами.

Формат:
- Згенеруй 60 варіантів.
- Кожен варіант: одне речення.
- Довжина: 8–18 слів.
- Без нумерації.
- Поверни результат у форматі JSON-масиву:
[
  { "text": "..." }
]

Додаткова умова:
- приблизно 70% текстів мають бути атмосферними й красивими
- приблизно 30% можуть бути легшими, теплішими або трохи жартівливими

Перед відправкою видали все, що:
- занадто прямо називає свято в кожному рядку
- звучить як реклама, привітання або тост
- повторює однакові образи
- занадто серйозне або занадто банальне
```
